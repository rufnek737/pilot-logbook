자주 묻는 질문

현재 서비스 동작과 정식 출시 정책을 기준으로 안내합니다.

현재 어떤 항공사를 지원하나요?

현재 CrewConnex 자동 가져오기와 ePaper 파싱은 제주항공 형식에 맞춰 제공됩니다. 다른 항공사는 실제 로스터 형식과 사용 권한을 확인한 뒤 순차적으로 지원할 예정입니다.

CrewConnex에서 어느 기간의 비행을 가져오나요?

가져오기 실행일을 기준으로 전후 5일 이내의 비행을 조회합니다. 조회 결과에서 적용할 비행을 직접 선택할 수 있습니다.

기존 비행을 다시 가져오면 어떻게 되나요?

같은 날짜와 편명의 기록을 찾아 신규·변경·동일로 구분합니다. 변경된 항목은 기존값과 CrewConnex 값을 비교한 뒤 사용자가 선택한 비행에만 적용됩니다. 사용자가 직접 작성한 기록은 보존됩니다.

CrewConnex 자동 가져오기 50편 이후에는 어떻게 되나요?

현재 시험 운영 기간에는 자동 가져오기 횟수 제한이 없습니다. 정식 유료 서비스가 시작되면 최초 50편의 자동 가져오기는 무료로 제공하고, 이후 자동 가져오기만 유료 기능으로 전환할 예정입니다. 유료화 시점과 가격은 적용 전에 앱에서 안내합니다.

결제하지 않아도 기존 기록을 계속 사용할 수 있나요?

네. 기존 비행기록 조회·수정, 수동 입력, 인쇄와 PDF 저장은 계속 이용할 수 있습니다. 결제하지 않았다는 이유로 기존 기록을 삭제하거나 잠그지 않습니다.

비행기록은 어디에 저장되나요?

비행기록과 기존 경력 수치는 기기 내부 저장소와 로그인 계정의 Firebase Firestore에 저장됩니다. 파싱을 위해 선택한 PDF·Excel·CSV 원본과 기존 경력에 첨부한 증빙 PDF·사진은 Pilot Logbook 서버에 업로드하지 않습니다. 경력 증빙 첨부파일은 선택한 휴대폰 안에만 저장됩니다.

기기를 변경하면 기록을 복원할 수 있나요?

동일한 계정으로 로그인하면 Firestore에 동기화된 비행기록과 기존 경력 수치를 불러올 수 있습니다. 경력 증빙 첨부파일은 클라우드 동기화 대상이 아니므로 기존 휴대폰의 ‘출력 → 휴대폰 이동용 전체 백업 → 백업 만들기’에서 백업 파일을 파일 앱·iCloud Drive·Google Drive에 보관한 뒤 새 휴대폰에서 ‘백업 복원’을 실행해야 합니다.

경력 증빙 첨부파일을 잃지 않으려면 어떻게 해야 하나요?

첨부파일은 선택한 기기에만 저장되며 서버에는 보관되지 않습니다. 앱 삭제, 휴대폰 초기화·분실·고장 전에 전체 백업 파일을 만들어 기기 밖의 안전한 위치에 보관해야 합니다. 백업하지 않은 기기 전용 첨부파일은 운영자가 복구할 수 없습니다.

회원 탈퇴 시 어떤 정보가 삭제되나요?

Firebase 로그인 계정, 클라우드 비행기록, 현재 기기의 로컬 비행기록과 해당 계정의 경력 증빙 첨부파일이 삭제됩니다. 삭제 후에는 복구할 수 없으므로 필요한 기록과 첨부파일을 먼저 전체 백업으로 보관해야 합니다.

이 앱의 기록을 공식 로그북으로 바로 제출할 수 있나요?

Pilot Logbook은 기록과 계산을 돕는 도구입니다. 항공사·관할 기관·채용기관이 요구하는 형식과 인정 기준은 다를 수 있으므로 제출 전 원본 자료와 관련 규정을 사용자가 직접 확인해야 합니다.
